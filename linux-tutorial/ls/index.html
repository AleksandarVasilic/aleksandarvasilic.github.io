Ls

This command is used for listing directory contents. If we run it we should get this result:

user@name:/$ ls
bin    dev   lib    libx32      mnt   root  srv  usr
boot   etc   lib32  lost+found  opt   run   sys  var
cdrom  home  lib64  media       proc  sbin  tmp

Additionally we can use different parameters that we can give it to the command, such as ls -l. If we man this command, -l parameter used for long listing format. Meaning it gives us more information about the content of the directory. If we run it in our root folder the following shall be listed:

user@name:/$ ls -l
total 72
lrwxrwxrwx   1 root root     7 Jul 20 17:27 bin -> usr/bin
drwxr-xr-x   4 root root  4096 Jul 27 12:53 boot
drwxr-xr-x   2 root root  4096 Jul 20 17:30 cdrom
drwxr-xr-x  20 root root  4620 Jul 29 11:31 dev
drwxr-xr-x 168 root root 12288 Jul 27 13:10 etc
drwxr-xr-x   4 root root  4096 Jul 20 17:31 home
lrwxrwxrwx   1 root root     7 Jul 20 17:27 lib -> usr/lib
...etc

Explaining the printed collumns

First column

Tells us the priveleges and file types of the content.

Ex. drwxr-xr-x first character of this sequence tells us that if the item is directory, file or link (d - directory, - - file, l - link). Other chartachter will be explained here.

Second column

Idicates how many memory locations/blocks directory/file has.

Third column

Tell us which name of the given file is owner of that file.

Fourth column

Tells us the group owner of the file.

In GNU/Linux systems besides the name of the users there are name of the groups which certain user can belong to. One user can be part of the one or more groups. Ex. if our user belongs to the root group, it could access the root files.

Fifth column

Tells us how much space in bytes given file types contain. Note! 4096 donen’t mean that the given directory with all its content contain 4 Kilobytes, just the given directory without its content contains contain 4K, but the content inside of it are independent files and directories that could contain other files and directories

Sixth column

Tells us the date of last change that has occured .

Seventh column

Tells us the name of the given file/directory.
