Cd

For now we will practise our cd command and get some usefull insites.

Make sure that you are in root directory, if not view this.

Next up we will see how to utilize relative and absolut paths. Type:

user@name:~$ cd etc/
user@name:/etc$

This was example of using relative path.

if we try to that again in our /etc/ folder we would be able to do it, because another etc folder does not exit in that direcotory.

user@name:/etc$ pwd
/etc
user@name:/etc$ cd etc/
bash: cd: etc/: No such file or directory

To futher solidify this we will change our directory to /home/user, and try to change our directory to /etc/ using relative path.

user@name:/etc$ cd /home/user/ # this is absolute path
# Or another very usefull way to change to our home directory is by running 
user@name:~$ cd ~
# From here we will try yo run 
user@name:~$ cd etc/
bash: cd: etc/: No such file or directory
# As we can see, we couldn't change our directory to the etc because the etc folder does not exist in /home/user
# Changing it back to /etc/
user@name:~$ cd /etc/
# This will work because there is a /etc/ folder inside / (root) directory.

Now we will run once again ls -l command to see content of the current directory.

user@name:/etc$ ls -l

... and other thousant files will be listed, yours might be more or less files listed

This will give you a enorumous list of files that you won’t be able to see on your screen. If you want to navigate up and down to see all printed stuff you can use Shift + PageUp/PageDown to scroll the screen.

Take a look of this:

lrwxrwxrwx  1 root root      19 Jul 20 17:35 mtab -> ../proc/self/mounts

Its a linked file that points to the relative path of ../proc/self/mounts. This ../ is saying to go back one directory up, ../../ would mean go two directoires up, ../../../ three, etc. Ex.

user@name:/etc$ cd /usr/share/zoneinfo/Europe/
user@name:/usr/share/zoneinfo/Europe/$ pwd
/usr/share/zoneinfo/Europe
# Now run cd ../ and print directory
user@name:/usr/share/zoneinfo/Europe/$ cd ../
user@name:/usr/share/zoneinfo/$ pwd
/usr/share/zoneinfo
# We can see that we have gone up one directory
user@name:/usr/share/zoneinfo/$ cd ../../../
user@name:/$ pwd
/
# now we are in root folder

clear

clear is the command that clears our terminal. Optionaly you can use Ctrl + l to perform quick clear of the terminal, but it behaves little differently than command clear depending on terminal emulators.
